iOS微信支付

  • 有几个库需要到进去，官方并没有说

    • TGRGETS->Build Phases->Link Binary With Libraries
    • 导入libc++.tbd(版本不同可能是libc++.dylib)
    • 导入libz.tbd
    • 导入libsqlite3.0.tbd

    统一下单(golang服务端)

  • 扫码支付有两种模式，根据不同的情况选择，主要说一下最重要的地方

  • 签名代码

  • 值得注意的是：回调的url必须不能带参数，不然微信异步给我们服务端发请求的时候回失败

支付宝

iOS Alipay

谈一下接官方SDK会遇到的问题吧

  • 问题1：Util/openssl_wrapper.m:11:9: ‘rsa.h’ file not found
    • 原因：官方SDK引入了pch文件(所谓pch文件就是标准的预编译头文件)，对应设置了Header Search Paths
    • 解决Build Settings->Header Search中 Paths添加$(PROJECT_DIR)/openssl和$(PROJECT_DIR)(或者添加一个$(SRCROOT)/项目的目录名字)
  • 问题2： Util/base64.h:63:21: Cannot find interface declaration for ‘NSObject’, superclass of ‘Base64’(一堆Unknown也这样解决就好)
    • 也是因为pch文件导致的(因为pch文件引入了uikit和Foundation框架)
    • 解决：直接引入Foundation就可以

  • 问题3：

    _CNCopyCurrentNetworkInfo”, referenced from:
    Undefined symbols for architecture x86_64:
    “*CNCopyCurrentNetworkInfo”, referenced from:
    -[APayReachability wifiInterface] in AlipaySDK
    +[internal_DeviceInfo getSSIDInfo] in AlipaySDK
    +[internal_DeviceInfo getNetworkInfo] in AlipaySDK
    “_CNCopySupportedInterfaces”, referenced from:
    -[APayReachability wifiInterface] in AlipaySDK
    +[internal_DeviceInfo getSSIDInfo] in AlipaySDK

    • 解决方案，导入一下几个库

Objective-c中__bridge

发表于 2016-08-15   |   分类于 Objective-c   |  

__bridge只做类型转换，但是不修改对象（内存）管理权；

__bridge_retained（也可以使用CFBridgingRetain）将Objective-C的对象转换为Core Foundation的对象，同时将对象（内存）的管理权交给我们，后续需要使用CFRelease或者相关方法来释放对象；

__bridge_transfer（也可以使用CFBridgingRelease）将Core Foundation的对象转换为Objective-C的对象，同时将对象（内存）的管理权交给ARC
